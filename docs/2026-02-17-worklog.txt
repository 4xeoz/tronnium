PROBLEMS WE FACED (and root causes)
- SSE cannot use request body (EventSource is GET-only).
  - Root cause: attempted to POST a body to an EventSource endpoint — browser does not permit it.
- EventSource does not send cookies/headers by default.
  - Root cause: SSE (EventSource) lacks header/credentials support so cookie-based JWTs are not sent.
- Authentication route hang (requests never returned).
  - Root cause: `passport.authenticate()` used without a callback would silently fail and not call `next()` (no response sent).
- Duplicate middleware / ordering confusion.
  - Root cause: applying `jwtAuthGuard()` multiple times or in wrong order caused unexpected behaviors.
- UI state and typing bugs (asset deletion UI not updating; missing imports; incorrect CpeCandidate object shape).
  - Root cause: asynchronous state not communicated to parent; TypeScript type mismatch and missing React imports.

WHAT WE TRIED (and what failed)
- Tried to stream SSE with a POST + JSON body — failed (EventSource doesn't support body).
- Attempted to secure SSE by relying on cookie auth alone — failed because EventSource doesn't include cookies from some cross-origin setups.
- Tried putting `jwtAuthGuard` on individual routes and also `router.use(jwtAuthGuard())` — led to redundancy and occasional middleware ordering issues.

FINAL SOLUTIONS IMPLEMENTED
- SSE
  - Exposed `/assets/cpe/find` as a GET SSE endpoint and moved it so it is reachable without cookie auth (or accept token via query param) to accommodate EventSource limitations.
  - `sendProgress` used across services so `findCpe` and ranking emit granular steps (parsing, search, scoring, formatting).
  - Frontend uses `listenForCpeFindProgress()` + `EventSource` to display live pipeline updates in the AddAsset slide-over.
- Authentication
  - Reworked `jwtAuthGuard()` to use a callback that sends 401/500 responses instead of leaving requests hanging.
  - Added logging to JWT strategy and cookie extraction to make token path and failures visible in logs.
- Relationship / Edges
  - Implemented relationship endpoints + service validation to enforce allowed `type` and `criticality` values.
  - React Flow: nodes/edges reflect backend relationships; connecting nodes triggers create; editing sidebar triggers update; deletion removes edge and database row.
- UX fixes
  - Asset deletion: parent list refreshed via callback after successful delete.
  - Fixed TypeScript typing issues (manual CPE candidate shape) and missing imports.

- Prefer server-side validation in services and keep controllers thin.
